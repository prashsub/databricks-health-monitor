# Spacing - Design System

## Overview

Spacing system for Databricks Health Monitor. Uses a 4px base unit for consistent, harmonious layouts. Optimized for data-dense monitoring interfaces.

## Spacing Scale

Based on 4px unit. Use these values consistently.

| Token | Value | Pixels | Usage |
|-------|-------|--------|-------|
| **0** | 0 | 0px | No spacing |
| **px** | 1px | 1px | Borders only |
| **0.5** | 0.125rem | 2px | Hairline gaps |
| **1** | 0.25rem | **4px** | Tight gaps, icon padding |
| **2** | 0.5rem | **8px** | Default component gap |
| **3** | 0.75rem | 12px | Medium gaps |
| **4** | 1rem | **16px** | Card padding, section gap |
| **5** | 1.25rem | 20px | Large component padding |
| **6** | 1.5rem | **24px** | Page margins, section spacing |
| **8** | 2rem | **32px** | Large section gaps |
| **10** | 2.5rem | 40px | Hero spacing |
| **12** | 3rem | 48px | Page section spacing |
| **16** | 4rem | 64px | Major section breaks |

### Quick Reference (Most Used)

```
4px  = xs  = tight gaps, icon padding
8px  = sm  = default gaps between items
16px = md  = card padding, standard spacing
24px = lg  = page margins, section gaps
32px = xl  = large section gaps
```

- Gap: 8px

- Alignment: Top Left

```

- Gap: 8px
- Padding: 0
- Alignment: Center

```

- Padding: 0
- Alignment: Stretch
